# SEO Strategy: Project Bodhi

## Strategic Overview

The website has a structural SEO advantage that most content sites never achieve: **775 individual, deeply-focused pages** (580 Vivekachudamani verses + 195 Yoga Sutras) targeting long-tail keywords with near-zero competition. This is a programmatic SEO goldmine.

**Primary Goal:** Become the #1 Google result for "vivekachudamani" and related terms within 6 months, and a top-3 result for broader Vedanta/Yoga Sutra queries within 12-18 months.

**Estimated Organic Traffic Potential:**
- Month 6: 20,000 monthly visitors
- Month 12: 100,000 monthly visitors
- Month 24: 300,000+ monthly visitors

## 2. Technical SEO Checklist for Next.js Site

### Crawling & Indexing

- [ ] **Dynamic sitemap generation** (`/sitemap.xml`): Auto-generate from verse/sutra database. Next.js App Router supports this natively via `app/sitemap.ts`
- [ ] **robots.txt** properly configured: Allow all public pages, block `/api/`, `/admin/`, preview deployments
- [ ] **Submit sitemap to Google Search Console** on launch day
- [ ] **Submit sitemap to Bing Webmaster Tools** (often overlooked; Bing drives 5-8% of search traffic)
- [ ] **Canonical URLs** on every page to prevent duplicate content (`<link rel="canonical">`)
- [ ] **No duplicate content between Hindi and English versions**: Use `hreflang` tags or separate URL paths (`/en/verse/1`, `/hi/verse/1`)
- [ ] **Pagination for verse lists**: Use proper `rel="next"` / `rel="prev"` or load-more patterns
- [ ] **404 page** with helpful navigation and search functionality
- [ ] **Redirect strategy**: If URLs ever change, implement 301 redirects

### Performance (Core Web Vitals)

- [ ] **Target scores:** LCP < 2.5s, FID < 100ms, CLS < 0.1
- [ ] **Static generation (SSG)** for all verse/sutra pages -- these are static content, no need for server-side rendering
- [ ] **Next.js Image Optimization** (`next/image`) for all images, especially quote cards and illustrations
- [ ] **Font optimization**: Self-host Devanagari fonts (Eczar, Vesper Libre) and Latin fonts (Cormorant Garamond). Use `next/font` for automatic optimization, `font-display: swap`
- [ ] **Lazy load below-fold content**: Audio players, related verses, comments
- [ ] **Minimize JavaScript bundle**: Use Next.js dynamic imports for heavy components (audio player, interactive Ashtanga wheel)
- [ ] **CDN caching via Vercel Edge Network**: Automatic with Vercel hosting
- [ ] **Compress images**: WebP format, appropriate sizing
- [ ] **Preconnect to external domains**: If using external fonts, analytics, or CDN

### On-Page SEO (Per Verse/Sutra Page)

- [ ] **Unique title tag** (50-60 chars): `Vivekachudamani Verse 1 - [Key Concept] | Project Bodhi`
- [ ] **Unique meta description** (150-160 chars): Compelling summary that includes the verse's key teaching
- [ ] **H1 tag**: One per page, contains primary keyword (verse number + key concept)
- [ ] **Structured heading hierarchy**: H1 > H2 > H3 (never skip levels)
- [ ] **Alt text on all images**: Descriptive, includes relevant keywords naturally
- [ ] **Open Graph tags** for social sharing: Custom OG image per verse (Sanskrit calligraphy card)
- [ ] **Twitter Card tags**: Large image card format
- [ ] **Clean URL structure**: `/vivekachudamani/verse/1`, `/yoga-darshan/samadhi-pada/sutra/1`
- [ ] **Breadcrumb navigation**: Home > Vivekachudamani > Chapter > Verse N

### Mobile Optimization

- [ ] **Mobile-first design**: 70%+ of spiritual content consumption is mobile
- [ ] **Touch-friendly navigation**: Large tap targets (48px minimum)
- [ ] **Readable font sizes**: 16px minimum body text, 20px+ for verse display
- [ ] **No horizontal scrolling**: Ensure Sanskrit text wraps properly on small screens
- [ ] **Fast mobile load**: Target < 3s on 3G connection
- [ ] **AMP not needed**: Next.js SSG with good Core Web Vitals is sufficient

### International SEO

- [ ] **`hreflang` tags** for Hindi and English versions of each page
- [ ] **Language selector** visible on all pages
- [ ] **Proper charset declaration** for Devanagari: `<meta charset="UTF-8">` (standard in Next.js)
- [ ] **Direction attributes**: `dir="ltr"` for both Hindi and English (Devanagari is LTR)

## 3. Internal Linking Strategy

### The Hub-and-Spoke Model

```
                    [Homepage]
                   /    |     \
          [Vivekachudamani]  [Yoga Darshan]  [Concepts]
           /    |    \        /    |    \       /   |   \
      [V.1] [V.2] [V.580] [S.1] [S.2] [S.195] [Maya] [Pancha Kosha] [8 Limbs]
```

### Linking Rules

**Rule 1: Every verse page links to related concept pages**
- Verse about Maya links to `/concepts/maya`
- Verse about Pancha Kosha links to `/concepts/pancha-kosha`
- Sutra about Yama links to `/concepts/8-limbs-of-yoga`

**Rule 2: Every concept page links to all relevant verses**
- `/concepts/maya` includes a section: "Maya in the Vivekachudamani" with links to every verse that discusses Maya
- This creates bidirectional linking between concept hubs and individual verse pages

**Rule 3: Verses cross-reference each other**
- If verse 152 elaborates on a concept introduced in verse 50, link them
- "Related Verses" section at the bottom of each page (3-5 related verses)
- This keeps users exploring and signals topical depth to Google

**Rule 4: Pillar articles link to specific verses as evidence**
- "What is Advaita Vedanta" article links to 10-15 specific verses that illustrate each concept
- "8 Limbs of Yoga" article links to the specific sutras for each limb

**Rule 5: Blog posts link back to verse pages and pillar articles**
- Every blog post should contain 3-5 internal links minimum
- Blog post about "self-inquiry" links to the "Self-Inquiry Practice Guide" AND to specific verses about self-inquiry

### Implementation

**Database Design for Cross-References:**
```
verse_relationships table:
  verse_id_a  |  verse_id_b  |  relationship_type
  VC-50       |  VC-152      |  "elaborates"
  VC-120      |  YD-1.2      |  "parallels"
  VC-200      |  VC-210      |  "continues"
```

**Tag/Theme System:**
Each verse tagged with themes: `maya`, `pancha_kosha`, `viveka`, `vairagya`, `atman`, `brahman`, etc.
- Auto-generate "Related Verses" based on shared tags
- Auto-generate concept page verse lists from tags

**Anchor Text Variety:**
Avoid using the same anchor text for every link. Rotate between:
- "Verse 152 explores this further"
- "As Shankaracharya explains in the section on Maya..."
- "See the Sanskrit root in verse 50"
- "This connects to Patanjali's second sutra"

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "What is Advaita Vedanta? A Complete Guide",
  "author": {

    "logo": {
      "@type": "ImageObject",
      "url": "https://projectbodhi.com/logo.png"
    }
  }

```

### 4B. Book Schema (for the source texts)

```json
{
  "@context": "https://schema.org",

  "alternateName": "The Crest-Jewel of Discrimination",
```

### 4C. FAQ Schema (on concept pages and verse pages with common questions)

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "What is the meaning of Vivekachudamani?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Vivekachudamani (विवेकचूडामणि) means 'The Crest-Jewel of Discrimination.' It is a philosophical poem of 580 verses by Adi Shankaracharya that teaches the path to Self-realization through discriminating between the Real (Brahman) and the Unreal (Maya)."
      }
    }
  ]
}
```

### 4D. BreadcrumbList Schema

```json
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "name": "Home", "item": "https://projectbodhi.com/" },
    { "@type": "ListItem", "position": 2, "name": "Vivekachudamani", "item": "https://projectbodhi.com/vivekachudamani" },
    { "@type": "ListItem", "position": 3, "name": "Verse 1", "item": "https://projectbodhi.com/vivekachudamani/verse/1" }
  ]
}
```

### 4E. HowTo Schema (for practice guides)

Use on pages like "How to Practice Self-Inquiry" or "Neti Neti Meditation Guide" to target featured snippets with step-by-step formatting.

### 4F. VideoObject Schema (for embedded YouTube videos)

When embedding verse explanation videos on verse pages, add VideoObject schema to enable video rich results in Google search.

### 4G. SpeakableSpecification (for voice search)

Mark the "modern interpretation" section of each verse as speakable content. As voice search grows, this positions content for Google Assistant and Alexa answers.

### Implementation in Next.js

Use `next-seo` package or manual `<script type="application/ld+json">` in page `<Head>`. Generate schema dynamically from the verse database:

```
// Pseudocode
For each verse page:
  Generate Article schema with verse-specific data
  Generate FAQ schema if verse has common questions
  Generate BreadcrumbList schema
  If video embedded: Generate VideoObject schema
```

## 8. Competitive SEO Gaps to Exploit

### Current SERP Analysis for Key Terms

**"vivekachudamani" (8,100/month):**
- Current top results: Wikipedia, Amazon book listings, archive.org PDF, a few low-quality blog posts
- No dedicated, well-designed, interactive digital edition exists
- Opportunity: A quality site can reach position 1-3 within 3-6 months

**"yoga sutras of patanjali" (18,100/month):**
- More competitive: yogapedia.com, yogajournal.com, Wikipedia
- But: no site offers the depth of individual sutra pages with Sanskrit + multi-language + modern interpretation
- Opportunity: Target long-tail sutra-specific queries first, build authority toward head terms

**"what is advaita vedanta" (9,900/month):**
- Current results: Wikipedia, Britannica, a few blog posts
- Most articles are surface-level (500-800 words)
- Opportunity: A 3,000+ word comprehensive guide with original verse references would outrank

**"pancha kosha" (4,400/month):**
- Thin content dominates: short blog posts, yoga studio pages
- No interactive, verse-backed explanation exists
- Opportunity: An interactive "5 Sheaths" guide with linked verses would easily rank #1

### The Structural Advantage

Most competitors have a handful of pages targeting these keywords. Project Bodhi will have 775+ deeply-focused pages creating a topical authority signal that is nearly impossible to replicate quickly. Google's algorithms heavily favor:

1. **Topical depth** -- 775 pages on Vivekachudamani/Yoga Sutras signals expertise
