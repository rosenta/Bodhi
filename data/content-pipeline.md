# Project Bodhi: Automated Content Generation Pipeline

## Overview

The content pipeline transforms raw verse data into platform-ready content across multiple formats, using Claude API for generation, n8n for orchestration, and Supabase for state management. The system is designed for a solo creator to maintain daily multi-platform publishing with roughly 30 minutes of daily review time.

---

## 1. How a New Verse Enters the System

### 1a. Bulk Import (Initial Setup)

```
yoga-sutras-complete.json ──> seed-yoga-sutras.ts ──> Supabase (verses table)
top-30-verses.json         ──> seed-data.sql       ──> Supabase (verses table)
```

- Run `seed-data.sql` for the initial 30 Vivekachudamani + 51 Samadhi Pada verses.
- Run `seed-yoga-sutras.ts` to programmatically insert remaining 144 Yoga Sutras.
- All verses enter with `is_published = true` and tags pre-populated from JSON.
- Theme mappings are auto-assigned based on pada number (Yoga Sutras) or JSON `theme` field (Vivekachudamani).

### 1b. Manual Addition (Ongoing)

When adding new Vivekachudamani verses beyond the initial 30:

1. **Admin Dashboard** (Next.js) or **Supabase Studio** - insert row into `verses` table.
2. Required fields: `text_id`, `verse_number`, `verse_order`, `sanskrit`, `english_translation`.
3. Optional enrichment fields (`modern_interpretation`, `reel_hook`, `practice_prompt`) can be left null and generated by the pipeline.
4. A database trigger (or n8n webhook) detects the new insert and queues enrichment.

### 1c. Verse Enrichment Flow

For verses missing optional fields, an enrichment sub-pipeline runs:

```
New verse detected (webhook)
    |
    v
Claude API: Generate missing fields
    ├── modern_interpretation (if null)
    ├── reel_hook (if null)
    ├── practice_prompt (if null)
    └── suggested tags (if empty)
    |
    v
Update verse row in Supabase
    |
    v
Mark verse as "enrichment_complete" (metadata flag)
```

**Claude Prompt Template for Enrichment:**

```
You are a wisdom teacher who bridges ancient Sanskrit philosophy with modern life.

Given this verse:
- Sanskrit: {sanskrit}
- English Translation: {english_translation}
- Theme: {theme}
- Text: {text_title}

Generate:
1. modern_interpretation: A 3-5 sentence interpretation for a modern audience
   aged 20-40. Use vivid metaphors. Reference modern life (phones, social media,
   careers) where it deepens the teaching. No spiritual jargon.
2. reel_hook: A single punchy sentence (under 15 words) that stops scrolling.
   Must create curiosity or pattern-interrupt.
3. practice_prompt: A specific 2-5 minute practice anyone can do today.
   Start with a concrete action verb.
4. tags: 3-6 lowercase keywords for discovery.
```

|-------|-------------------------|

### Content Status Flow

```
draft ──> in_review ──> approved ──> published
  │                        │
  └── (rejected) ──────────┘ (re-generated)
                             │
                         archived
```

- **draft**: AI-generated, awaiting human review.
- **in_review**: Flagged for creator review (15-30 min daily task).
- **approved**: Ready to publish at scheduled time.
- **published**: Live on platform.
- **archived**: Taken down or superseded.

## 4. Analytics Feedback Loop

### Data Collection

```
Platform APIs (Instagram, YouTube, Twitter)
    │
    ▼
n8n scheduled job (weekly, Sunday night)
    │
    ▼
Fetch metrics for all published content_pieces
    │
    ▼
Insert into content_performance table:
    - impressions, reach, likes, comments,
      shares, saves, clicks, watch_time_s
    │
    ▼
Aggregate and store in raw_data (jsonb) for
platform-specific metrics
```

### Metrics Tracked

| Metric | Source | Used For |
|--------|--------|----------|
| Impressions | Platform API | Reach analysis |
| Engagement rate | likes + comments + shares / impressions | Content quality signal |
| Save rate | saves / impressions | "Deep value" indicator |
| Watch time | YouTube/Instagram | Reel effectiveness |
| Click-through | Publer / UTM links | Website traffic attribution |
| Verse views | Internal analytics | Most popular verses |
| Search queries | Internal search logs | Content gap analysis |
| Bookmark count | Supabase | User favorites signal |

### Feedback into Content Strategy

#### Weekly Analysis (automated n8n workflow)

```
Every Sunday at 10 PM:

[1] Pull content_performance for the past 7 days
    │
[2] Claude API: Analyze performance data
    │   - Which themes performed best?
    │   - Which content types got most engagement?
    │   - Which platforms drove the most traffic?
    │   - Which reel hooks stopped the most scrollers?
    │
[3] Generate recommendations:
    │   - Adjust daily_verse schedule for next week
    │   - Suggest which themes to lean into
    │   - Flag underperforming content types
    │   - Recommend A/B test variants
    │
[4] Store analysis in a weekly_report table (or send to admin email)
```

#### Long-Term Optimization Signals

| Signal | Action |
|--------|--------|
| A theme consistently gets 2x engagement | Schedule that theme more frequently |
| A content_type underperforms across all verses | Reduce frequency or retire the format |
| Certain reel_hook patterns outperform | Train Claude prompts with those patterns |
| Verse views cluster on specific verses | Create "deep dive" series on those verses |
| High bookmark rate but low share rate | Content is valued but not viral — adjust CTA |
| Search queries with no matching verses | Signal to add new verses or create explainer content |

### A/B Testing Framework

For reel hooks (the highest-leverage content element):

1. Generate 3 hook variants per verse via Claude API.
2. Publish variant A on Day 1, variant B on Day 2 (same verse, different hook).
3. Compare 48-hour engagement metrics.
4. Store winning hook pattern in a `hook_patterns` reference table.
5. Feed winning patterns back into Claude prompts as few-shot examples.

## 5. Technical Implementation

### n8n Workflow Nodes

```
Workflow: daily-content-pipeline
  Trigger: Cron (0 5 * * *)
  Nodes:
    1. HTTP Request → Supabase REST: GET daily_verse + verse
    2. HTTP Request → Claude API: Generate content (JSON mode)
    3. Code → Parse Claude response
    4. HTTP Request → Supabase REST: INSERT content_pieces (x6)
    5. HTTP Request → Canva API: Generate visuals (parallel)
    6. HTTP Request → ElevenLabs API: Generate audio (parallel)
    7. HTTP Request → Cloudflare R2: Upload assets
    8. HTTP Request → Supabase REST: UPDATE media_urls
    9. HTTP Request → Vercel API: Publish blog post
    10. HTTP Request → Publer API: Schedule social posts
    11. HTTP Request → Supabase REST: UPDATE status + scheduled_at
```

### Error Handling

| Failure Point | Recovery Strategy |
|---------------|-------------------|
| Claude API timeout/error | Retry 3x with exponential backoff. If all fail, skip and alert admin. |
| Canva API failure | Use fallback plain-text quote card template. |
| ElevenLabs failure | Skip audio; publish text-only. Queue for manual recording. |
| R2 upload failure | Retry 3x. If fail, store locally and retry on next pipeline run. |
| Publer scheduling failure | Log error. Admin manually schedules from dashboard. |
| Supabase write failure | Critical path: halt pipeline, alert admin immediately. |

## 6. Content Calendar Seeding Strategy

### Phase 1: First 30 Days (Launch)

Use the 30 pre-enriched Vivekachudamani verses alternating with the most impactful Yoga Sutras. Every verse already has `modern_interpretation`, `reel_hook`, and `practice_prompt` filled in.

### Phase 2: Days 31-195

Progressively work through all 195 Yoga Sutras (already fully loaded). Generate enrichment fields via the pipeline for each verse as it approaches its scheduled date.

### Phase 3: Days 196-580+

Expand Vivekachudamani beyond the initial 30 verses. Each new verse gets:
1. Manual entry of `sanskrit` + `english_translation` (from existing translations).
2. Automated enrichment via Claude API.
3. Automated content generation via daily pipeline.

### Phase 4: Repeat Cycles

After exhausting all 775 verses (580 + 195), begin themed repeat cycles:
- "Deep Dive" series: re-visit popular verses with new angles.
- "Practice Series": focus only on `practice_prompt` for 30 days.
- "Modern Parallels": pair each verse with a current event or trend.
- User-requested verses: let the community vote on which verse to feature next.

This gives 2+ years of unique daily content, followed by infinite themed variations.
